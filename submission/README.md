# Submission artefacts

This folder holds **what you hand in** (or export for Turnitin) and **school paperwork copies**, separate from the runnable codebase under `src/` and `scripts/`.

## Typical files

| Item | Role |
|------|------|
| `Arka_Talukder_Dissertation_Final.docx` | Generated by `python scripts/dissertation_to_docx.py` from `Dissertation_Arka_Talukder.md` |
| `Arka_Talukder_Dissertation_Final_Submission.docx` | Often the **final edited** filename after front-matter, TOC, and declaration pages are completed in Word |
| `Arka_Talukder_Dissertation_Final_Submission_Humanized_version.docx` | Optional **humanized** track if you maintain `Dissertation_Arka_Talukder_Humanized.md` |
| `forms/` | Moodle / school templates: front sheet, library form, final-report guideline, signed **project specification** |

## Sync workflow

- After editing the main thesis Markdown, run:

  ```bash
  python scripts/sync_dissertation_and_docx.py
  ```

  That regenerates the Word file here (when possible) and refreshes `supervisor_package/01_Dissertation/` if that directory exists.

- For humanized-only regeneration:

  ```bash
  python scripts/sync_humanized_md_and_docx.py
  ```

## Source of truth

- **Text and structure:** `Dissertation_Arka_Talukder.md` at the **repository root** (figure paths stay `assets/...` and `results/...`).
- **Canonical process/attendance appendices** embedded by the exporter: `archive/process_attendance/` (see `archive/README.md`).
